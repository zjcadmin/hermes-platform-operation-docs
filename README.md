# Hermes 通用平台操作文档生成提示词

这是一个面向 Hermes 自动化执行代理的通用提示词，用于辅助生成 Web 管理平台、业务系统、政务平台、医院信息平台、数据中台等系统的操作文档。

提示词核心文件：

- `Hermes_平台操作文档提示词.md`

它的目标是让 Hermes 在已登录的平台中，模拟真实用户逐级点击菜单、识别页面功能、打开按钮对应的弹窗或详情页、完成截图和文字说明，最终生成一份带 Word 自动目录、章节编号、截图和操作说明的系统操作文档。

## 适用场景

适合用于以下类型系统的操作文档编写：

- Web 管理后台
- 医院信息化平台
- 政务业务系统
- 数据中台或技术中台
- 运维管理平台
- 低代码、表单、流程、配置类后台
- 其他具有菜单树、列表页、按钮操作、弹窗表单的业务系统

## 核心能力

该提示词重点约束 Hermes 完成以下工作：

- 先建立完整菜单台账，再按菜单顺序逐项执行。
- 自动识别一级、二级、三级及更深层级菜单。
- 对最底层功能菜单截图并写说明。
- 识别查询区、列表区、工具栏、操作列、分页区、弹窗和按钮。
- 对新增、修改、配置、设置、授权、查看、详情、启用、停用、导入、导出等按钮分别处理。
- 对需要先选中列表数据的操作，先用视觉能力判断，再安全选中一条可见数据。
- 截图前等待页面加载完成，避免截到转圈、Loading、骨架屏或未渲染完成的页面。
- 截图前检查是否有命令行、文件夹、通知、下载栏等非平台窗口遮挡。
- 弹窗截图后必须点击取消、关闭、返回或叉号退出。
- 删除按钮不点击、不勾选、不打开确认框，只截图说明风险。
- 最终生成 Word 自动目录、截图清单、异常清单和审查记录。

## 安全边界

提示词内置了严格的安全边界，避免 Hermes 在真实系统中误操作数据：

- 不点击保存、提交、确定、确认等最终生效按钮。
- 不点击删除按钮，不打开删除确认框。
- 不执行启用、停用、发布、撤回、审核、同步、重跑等最终确认动作。
- 修改、配置、授权类操作只打开界面截图，截图后取消或关闭。
- 如果无法判断某个按钮是否会直接生效，则只截图说明，不点击。

## 使用方式

1. 打开 `Hermes_平台操作文档提示词.md`。
2. 将其中内容复制给 Hermes。
3. 在任务上下文中补充目标平台地址、登录状态、输出目录或特殊要求。
4. 确认目标平台已经登录，或说明需要 Hermes 复用哪个浏览器会话。
5. 让 Hermes 按提示词执行菜单遍历、截图和文档生成。
6. 执行完成后，检查 Hermes 输出的 Word 文档、截图目录、菜单台账和异常清单。

示例任务描述：

```text
请使用这个提示词，对当前已登录的业务系统生成操作文档。
平台地址为：http://example.local/#/index
默认输出到 E 盘当前任务目录。
注意不要执行任何保存、删除、启用、停用、发布、确认类操作。
```

## 推荐输出物

Hermes 执行完成后，建议至少保留以下文件：

- `系统操作文档.docx`
- `screenshots/` 截图原图目录
- `menu-ledger.xlsx` 或 `menu-ledger.csv` 菜单台账
- `screenshot-list.xlsx` 或 `screenshot-list.csv` 截图清单
- `exceptions.md` 异常清单

## 适配建议

不同平台的菜单结构、按钮命名和交互方式可能不同。使用前可以按需调整：

- 平台名称和文档标题。
- 输出目录。
- 是否需要截图所有按钮。
- 是否允许点击导出、下载模板等低风险按钮。
- 是否需要生成 Word、Markdown 或同时生成两种格式。
- 是否需要额外生成菜单台账、按钮清单或测试记录。

本提示词用于辅助生成系统操作文档。实际执行时应遵守目标系统的数据安全、隐私保护和运维管理要求。对于生产系统，建议使用只读账号或测试环境执行，避免误操作真实业务数据。
